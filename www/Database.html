LingTu AnEditor Project

Overview

Name: AnEditor
Platform: Windows Operation System
Language: C++, MFC, Socket, STL, COM, DCOM, SQL, sqlplus
Development Tool: Visual Studio 2010, Oracle SQL Developer
Database: Oracle 11g
Description: AnEditor is the product name, OptimizationDLL is a component of AnEditor which was developed by me
Functions:
  1. Generate global id on server side and distribute it to the client which requests. Note: there may be hundreds of clients request global id simultaneously
  2. Transfer a pretty long SQL sentence(for example: 8Mbytes long) to a shorter one
  3. Make use of materialized view to improve performance of joint query
  4. Develop triggers and stored procedures to optimize query speed

Functionalities

Global ID Distribution
  1. Server application listen port, be ready to accept request from client application
  2. Use block model socket in server application, then all requests from client applications must stand in line to wait for distributing global id
  3. Generate a global id. Get the max id value in globalid table in oracle database, then plus one to generate a new global id
  4. Record the new generated global id in globalid table in oracle database
Solution for long SQL sentence
  1. configure local Oracle service to remote execute sql via sqlplus (configuration document)
  2. Connect remote oracle database via sqlplus (connection script)
  3. Replace the long sql like "select * from table1 where id in(1,2,...,9999999999)" with a shorter sql like "select * from table1 where id in(select id from temptable)"
  4. Run a sql to create a temp table via sqlplus in client application
  5. Launch multiple processes to insert id(1,2,...,9999999999) into temp table via sqlplus in client application
  6. Remove temp table after using
Materialized view
  1. Connect remote oracle database via Oracle SQL Developer

  2. Create materialized view (script download)
  3. When application conducts joint query in Oracle database, it will query the materialized view instead. The behavior is equivalent with querying one table
Trigger & Stored procedure
  1. Set up some triggers to maintain the completeness of database (trigger)
  2. Client application call stored procedure to create/drop temp table, create/drop view in oracle database (stored procedure script)
  3. Client application call stored procedure to insert or update records in table (stored procedure script)
